---
name: Track all changes to avoid circular edits
description: Вести историю изменений в памяти, не добавлять/убирать один и тот же функционал по кругу
type: feedback
---

Вести полную историю своих изменений в файлах changelog в памяти.

**Why:** Без истории новая сессия может откатить или заново добавить то, что уже было сделано/убрано, теряя время на хождение по кругу.

**How to apply:** Перед любыми правками в ключевых файлах (server.ts, bus.py и т.д.) — прочитать соответствующий changelog из памяти. После правок — обновить changelog.
